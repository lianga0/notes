## C++中容器迭代器

简单来说，C++ STL 迭代器是连接容器（数据）和算法（操作）的“桥梁”。你可以把它想象成一个智能化的游标。

### 迭代器到底是什么

在设计模式中，迭代器是一种行为抽象。它的存在是为了让你无论在使用 `vector`、`list` 还是 `set` 时，都能用同样的方式（比如 `++` 或 `*`）去访问元素，而不需要关心底层是连续内存还是链表节点。

### `begin()` 和 `end()` 返回的是真实元素吗？

这两个函数返回的是迭代器对象，但它们指向的位置大有讲究：

`begin()`: 指向容器中的第一个真实元素，也就是直接指向第 `0` 个位置。只有当容器非空时，该位置才对应一个真实的元素；若容器为空，它与 `end()` 语义相等，不可解引用。

`end()`: 指向容器最后一个元素之后的那个位置。这是一个“哨兵”位，并不存在真实元素。

这就好比你去排队买票，`begin` 是排在第一位的人。如果根本没人排队，那么“排在第一位的人”这个位置依然存在，只是那个位置上是空的（此时`begin()`与 `end()` 语义相等）。

如果容器为空，`begin()` 和 `end()` 返回的是同一个位置，且这个位置不包含任何真实元素，`begin()` 此时等同于 `end()`。在 C++ 标准中，对 `end()` 迭代器或空容器的 `begin()` 进行解引用属于 未定义行为（Undefined Behavior）。这可能会导致程序崩溃（Segmentation Fault）或者读取到垃圾数据。

专业的 C++ 开发者在对迭代器解引用前，通常会做如下检查：

```
if (it != v.end()) {
    auto value = *it; 
}
```

> 注意：迭代器对象本身永远是真实存在的（它是一个占内存的变量），但它指向的内容在空容器状态下是不存在的。
